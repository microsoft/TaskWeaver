Quick Start

Installationâ€‹

You can install TaskWeaver by running the following command:

# [optional] create a conda environment to isolate the dependencies
# conda create -n taskweaver python=3.10
# conda activate taskweaver

# clone the repository
git clone https://github.com/microsoft/TaskWeaver.git
cd TaskWeaver
# install the requirements
pip install -r requirements.txt

Project Directoryâ€‹

OpenAI Configurationâ€‹

Before running TaskWeaver, you need to provide your OpenAI API key and other necessary information. You can do this by editing the taskweaver_config.json file. If you are using Azure OpenAI, you need to set the following parameters in the taskweaver_config.json file:

Azure OpenAIâ€‹

{
"llm.api_base": "https://xxx.openai.azure.com/",
"llm.api_key": "your_api_key",
"llm.api_type": "azure",
"llm.api_version": "the api version",
"llm.model": "the model name, e.g., gpt-4" # In Azure OpenAI, the model name is the deployment_name
}

OpenAIâ€‹

{
"llm.api_key": "the api key",
"llm.model": "the model name, e.g., gpt-4"
}

ðŸ’¡ Only the latest OpenAI API supports the json_object response format. If you are using an older version of OpenAI API, you need to set the "llm.response_format"=null in the taskweaver_config.json file.

More configuration options can be found in the configuration documentation.

Start TaskWeaverâ€‹

# assume you are in the TaskWeaver folder cloned from the repository
python -m taskweaver -p ./project/ # -p is the path to the project directory

This will start the TaskWeaver process and you can interact with it through the command line (CLI) interface. If everything goes well, you will see the following prompt:

There are other ways to start TaskWeaver:

  • A Chainlit UI interface: TaskWeaver provides an experimental web-based interface to interact with the system.
  • A Library: You can also use TaskWeaver as a library in your Python code.
  • The all-in-one Docker image: We provide a Docker image that contains all the dependencies to run TaskWeaver.
